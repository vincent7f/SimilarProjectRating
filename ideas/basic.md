# 目标

- 从专业软件人员的角度，实现一个能分析和比较github.com上具有类似功能的、开源项目的区别，这些区别包括：功能上、成熟度上、代码质量、网上评价等会影响用户选择的因素(这些因素将以github.com上最受欢迎的top 50项目为基准)，并给出一个综合的、多维度排序，再根据这些数据给出AI对用户选择的建议。

- 考虑到下载速度和成熟度，只下载该项目的最后一个release版本的代码，并对其进行分析，如果没有release版本，刚下载最新版本。

- 使用python实现，并使用github actions进行自动化测试。

- 用户输入关键词，交给AI生成多组适合github.com搜索的keyword组，再通过这些keyword组在github.com上搜索出相关项目，然后将这些项目资料提交给AI分析，过滤掉不相关的项目，只对剩下的项目进行分析比较。如果剩下的项目数量过多，刚只保底star数最多的前20个项目。

- AI model先使用本地Ollama上的模型，同时保留兼容openai接口和litellm的能力。

# Rules

- 每一步操作都要有详细的日志记录，并保存到本地，方便后续分析。

- 保存每次CodeBuddy的运行日志，方便后续分析。

- 每次运行后，总结这次执行的结果，将这次的成功和失败经验都总结和保存下来，并给出下一次执行的建议。

- 每一步都能自动提交到github.com上。

- 采用git worktree进行并行开发。

- 保留用户每次的与CodeBuddy的对话记录，方便后续分析。

- 编写文档时，以中语文为主，英文为辅，文档采用一段中文，再接一段英文（上段中文对应的英文）。

- 代码提交时，只能是用英文，禁止使用其它语言。
